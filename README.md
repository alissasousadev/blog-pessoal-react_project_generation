<h2> Sobre o projeto</h2>

<p>
  O <strong>Blog Pessoal Frontend</strong> é a interface web da aplicação de blog desenvolvida como parte prática do bootcamp da Generation.
  O projeto foi construído para consumir a API do Blog Pessoal, permitindo a interação entre usuário e sistema por meio de telas,
  componentes e fluxos de navegação típicos de uma aplicação SPA.
</p>

<p>
  A proposta deste front-end é oferecer uma experiência visual organizada e funcional, conectando autenticação,
  navegação e gerenciamento de conteúdo em uma interface construída com React.
</p>

<hr />

<h2> Objetivos de aprendizagem</h2>

<ul>
  <li>Desenvolver interfaces com React</li>
  <li>Praticar componentização e reaproveitamento de código</li>
  <li>Consumir dados de uma API REST</li>
  <li>Trabalhar integração entre front-end e back-end</li>
  <li>Organizar rotas, estados e fluxo de navegação da aplicação</li>
  <li>Aplicar boas práticas de estruturação de projeto front-end</li>
</ul>

<h2> Funcionalidades do front-end</h2>

<p>
  O projeto foi desenvolvido para atuar como camada de apresentação da aplicação Blog Pessoal,
  integrando a interface com os recursos disponíveis no back-end.
</p>

<ul>
  <li>Renderização de páginas e componentes da aplicação</li>
  <li>Navegação entre telas</li>
  <li>Integração com a API do Blog Pessoal</li>
  <li>Exibição e manipulação de dados retornados pelo back-end</li>
  <li>Estrutura preparada para autenticação e gerenciamento de conteúdo</li>
  <li>Atualização visual e interação com o usuário em tempo real</li>
</ul>

<hr />

<h2> Integração com o back-end</h2>

<p>
  Este projeto funciona em conjunto com a API do Blog Pessoal, responsável pelas regras de negócio,
  autenticação, persistência de dados e disponibilização dos endpoints consumidos pela aplicação front-end.
</p>

<p>
  O front-end envia requisições para a API e renderiza as informações recebidas, tornando a experiência do usuário
  mais dinâmica e intuitiva.
</p>

<hr />

<h2> Estrutura geral do projeto</h2>

<p>
  A estrutura do projeto foi organizada para separar arquivos estáticos, componentes e lógica da aplicação.
</p>

<ul>
  <li><code>public/</code> → arquivos públicos e estáticos</li>
  <li><code>src/</code> → código-fonte principal da aplicação</li>
  <li><code>index.html</code> → arquivo base da aplicação</li>
  <li><code>package.json</code> → dependências e scripts do projeto</li>
  <li><code>tsconfig.json</code> → configuração do TypeScript</li>
</ul>

<h2> Como executar o projeto localmente</h2>

<ol>
  <li>Clone este repositório</li>
</ol>

<pre><code>git clone https://github.com/alissasousadev/personal-blog-frontend.git</code></pre>

<ol start="2">
  <li>Acesse a pasta do projeto</li>
</ol>

<pre><code>cd personal-blog-frontend</code></pre>

<ol start="3">
  <li>Instale as dependências</li>
</ol>

<pre><code>npm install</code></pre>

<ol start="4">
  <li>Execute o projeto</li>
</ol>

<pre><code>npm run dev</code></pre>

<ol start="5">
  <li>Abra o endereço exibido no terminal, normalmente:</li>
</ol>

<pre><code>http://localhost:5173</code></pre>
